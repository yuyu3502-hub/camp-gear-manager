# 作業メモ

このリポジトリは、お父さんがキャンプ用品をトランクルーム内の保管場所と持ち出しセットで管理する静的Webアプリです。

## 優先する判断

- 最初の状態は、保管場所・用品・セットがすべて0件。
- 元から用品や保管場所が入っているように見せない。
- 例は入力欄のplaceholderやREADMEだけに置く。
- 「コンテナ」ではなく「保管場所」「トランクルーム内の部屋・棚・区画」を使う。
- 主な利用者はお父さん。現地で急いで登録できる文言を優先する。
- 最初はケース・バッグ単位で仮登録できる導線にする。
- 画面文言は、IT用語より操作結果が分かる言葉にする。
- 写真はIndexedDB、用品などのメタデータはlocalStorage。

## 主要ファイル

- `index.html`: 画面構造と文言。
- `styles.css`: レイアウトとレスポンシブ対応。
- `app.js`: 状態管理、保存、写真、描画、イベント処理。
- `README.md`: 利用者向け説明。
- `scripts/verify.sh`: 変更後の軽い確認。

## 変更後の確認

最低限、以下を実行する。

```bash
./scripts/verify.sh
```

画面やスマホ表示を変えた場合だけ、ブラウザで追加確認する。

```bash
python3 -m http.server 8790
```

## 公開

GitHub Pagesは `main` ブランチ直下から配信。

```bash
git push origin main
```

公開URL:

```text
https://yuyu3502-hub.github.io/camp-gear-manager/
```

## 次回作業の進め方

- まず `AGENTS.md` と該当ファイルだけ読む。
- 変更範囲が文言だけなら、全ファイル探索やブラウザ自動操作は省く。
- デプロイ前は `./scripts/verify.sh` を通す。
- GitHub Pages反映確認は、必要な文言だけ `curl` で確認する。
